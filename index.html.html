Important Notes
⚠️ Before You Proceed
  • Maintenance mode will temporarily disable user access to the system.
  • Always inform users or stakeholders before enabling maintenance mode.
  • Ensure database connectivity and all required services are stable before switching back to live mode.
  • Verify Docker container status before executing maintenance commands.
Verification
Check Running Containers
sudo docker compose ps
Inspect Logs (last 50)
sudo docker compose logs --tail=50
Pre-Live Checklist
  • Containers are running properly
  • No application errors in logs
  • Database connections are healthy
  • Web services respond correctly
General Workflow
01
Navigate to System Directory
Use cd to change into the target system's path before running any commands.
02
Enable Maintenance Mode
Run php artisan down via Docker Compose to take the system offline for users.
03
Perform Required Maintenance
System updates · Deployment · Database migration · Configuration changes · Service restart
04
Verify System Health
Check container status with docker compose ps and review logs for errors.
05
Return System to Live Mode
Run php artisan up to restore user access once all checks pass.
